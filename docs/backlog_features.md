# Phase 9: Growth & Engagement 백로그 (성장 & 바이럴 엔진)

이 문서는 Shorts100.com MVP 출시 이후 사이트 활성화 및 크리에이터 유입을 가속화하기 위한 성장 기능들의 상세 기획 및 구현 가이드입니다. 

현재 **DB 스키마 및 SQLAlchemy 모델 수준의 뼈대(Skeleton) 설정이 완료**되어 있어, MVP 오픈 이후 언제든지 즉시 세부 기능을 구현할 수 있는 상태입니다.

---

## 📋 핵심 성장 기능 & 뼈대 현황

### 1. 크리에이터 명예의 전당 알림 및 자동 인증서 (Viral Engine)
* **목적**: TOP 100에 진입한 쇼츠 크리에이터들에게 알림을 발송하고, 자신의 SNS(유튜브 커뮤니티, 인스타그램 등)에 공유할 수 있는 동적 인증서 이미지를 제공하여 외부 트래픽을 유입합니다.
* **DB 뼈대 구성 완료**:
  * `channels.owner_email` (이메일 수신용 주소)
  * `channels.notify_on_rank` (랭킹 진입 시 이메일 수신 동의 여부)
  * `channels.verification_token` (소유자 인증용 토큰)

#### 🛠️ 향후 구현 단계 (Phase 9)
1. **소유권 인증 및 알림 신청 폼 (Frontend)**
   * 채널 상세 페이지 또는 하단에 "이 채널의 소유자이신가요? 알림 받고 채널 인증하기" 영역 노출.
   * 이메일 주소를 입력받아 `verification_token`을 포함한 메일 발송 및 인증 완료 시 `owner_email` 매핑.
2. **동적 인증서 생성 API (Frontend - Next.js `/api/og/cert`)**
   * `@vercel/og` 라이브러리를 활용하여 채널명, 쇼츠 제목, 순위, 날짜 등을 결합한 고해상도 인증 이미지 동적 생성.
   * 예: `/api/og/cert?channel_id=123&rank=5&date=2026-05-22`
3. **랭킹 알림 Celery Task (Backend)**
   * 매시간 랭킹 연산 (`compute_rankings` 또는 차트 스냅샷) 완료 후, `notify_on_rank=True`인 채널 중 새로 TOP 10에 진입하거나 이전 대비 순위가 상승한 채널을 스캔.
   * 등록된 `owner_email`로 축하 이메일 및 인증서 이미지 링크 자동 발송 (SendGrid 또는 AWS SES 연동).

---

### 2. 익명 이모지 반응 버튼 (Engagement Loop)
* **목적**: 사용자들이 가볍게 탭하여 감정 표현(🔥, 🤩, 💯, 🤔 등)을 할 수 있도록 유도하여 체류 시간과 참여도를 높입니다.
* **DB 뼈대 구성 완료**:
  * `video_reactions` 테이블 신설 완료.
  * 중복 반응 방지를 위해 `(video_id, session_id)` 유니크 제약 조건 지정.

#### 🛠️ 향후 구현 단계 (Phase 9)
1. **반응 버튼 UI (Frontend)**
   * 쇼츠 상세 페이지 또는 메인 리스트 카드 우하단에 이모지 반응 버튼 모음 제공.
   * 사용자가 클릭하면 로컬 스토리지 또는 쿠키에 `session_id`를 발송해 중복 클릭 방지.
2. **반응 추가/조회 API (Backend)**
   * `POST /api/videos/{video_id}/reactions`: 이모지 및 `session_id` 전달받아 DB에 기록.
   * `GET /api/videos/{video_id}/reactions`: 현재 이모지별 누적 개수 반환 (캐싱 적용).
3. **랭킹 알고리즘 가중치 연동 (Advanced)**
   * Z-score 연산 시 단순 조회수 외에 24시간 내 발생한 이모지 반응 수에 가중치를 부여하여 실시간 급상승(Rising) 차트 정확도 개선.

## 💡 개발 가이드 및 팁
* **이메일 발송**: 알림 이메일은 Celery 비동기 태스크로 처리하여 API 응답 속도에 영향을 주지 않도록 설계해야 합니다.
* **인증서 렌더링**: 동적 OG 이미지 생성 시 유튜브 썸네일과 채널 아바타 이미지를 Next.js가 렌더링하도록 `next.config.mjs` 리모트 패턴 설정을 확인하십시오.
* **쿠키 & PIPA**: 익명 반응에 사용되는 `session_id`는 개인 식별이 불가하도록 무작위 UUID 형태로 발급하고, 필요시 쿠키 안내 문구(`consent_log`)와 연계하십시오.
